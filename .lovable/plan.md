
## الهدف
نسخ كامل كود مشروع `orlyxhq/-Al-Nahda-Company` من GitHub إلى مشروع Lovable الحالي، ليصبح المشروع جاهزاً للعمل والتعديل عليه هنا.

## السياق
- المستودع المصدر: https://github.com/orlyxhq/-Al-Nahda-Company (عام)
- المستودع هو في الأصل مشروع Lovable بنفس البنية التقنية (TanStack Start + React 19 + Vite + Tailwind v4 + Bun)
- المشروع الحالي يحتوي فقط على القالب الافتراضي (صفحة فارغة)
- بما أن البنية متطابقة، النسخ سيكون مباشراً دون الحاجة لتحويلات

## الخطوات

### 1. جلب قائمة كامل الملفات من GitHub
- استخدام GitHub API لجلب شجرة الملفات الكاملة للفرع `main`
- تصفية الملفات التي ستُنسخ (تجاهل `.git/`، `node_modules/`، `bun.lock` الكبير إذا تعارض)

### 2. تنزيل ومزامنة الملفات
نسخ المجلدات والملفات التالية من المستودع إلى المشروع الحالي:
- `src/` بكامل محتوياتها (المكونات، المسارات، الأصول، الأنماط)
- `package.json` (لتثبيت التبعيات الصحيحة)
- `tsconfig.json`، `vite.config.ts`، `components.json`
- `.prettierrc`، `.prettierignore`، `eslint.config.js`، `bunfig.toml`
- `AGENTS.md`
- مجلد `.lovable/` إن كان يحتوي على إعدادات مهمة

سيتم استبدال الملفات الموجودة حالياً (مثل `src/routes/index.tsx` الفارغة) بالمحتوى من المستودع.

### 3. تثبيت التبعيات
- تشغيل `bun install` لتثبيت كل الحزم المذكورة في `package.json` الجديد

### 4. فحص ما بعد النسخ
- التأكد من أن `src/routes/__root.tsx` و `src/router.tsx` موجودان وسليمان
- التحقق من بناء المشروع بنجاح
- معاينة المشروع للتأكد من أنه يعمل كما هو على GitHub

### 5. الإبلاغ بالنتائج
- إخبارك بعدد الملفات المنسوخة
- الإشارة إلى أي ميزات تحتاج إعداد إضافي (مثل Lovable Cloud إذا كان المشروع الأصلي يستخدم قاعدة بيانات/مصادقة)

## تفاصيل تقنية
- سأستخدم GitHub API (`api.github.com/repos/.../git/trees/main?recursive=1`) للحصول على قائمة الملفات
- ثم `curl` لتنزيل المحتوى الخام عبر `raw.githubusercontent.com`
- الملفات النصية تُكتب عبر أدوات الكتابة، والصور/الأصول الثنائية عبر تنزيل مباشر
- بعد النسخ، Vite dev server سيُعاد تشغيله تلقائياً بعد `bun install`

## ما لن يتم في هذه الخطة
- لن أعدّل على تصميم أو وظائف المشروع الأصلي — فقط نسخ كما هو
- لن أُفعّل Lovable Cloud تلقائياً (سنرى أولاً ما يحتاجه المشروع بعد النسخ)
- أي تعديلات تريدها على المحتوى ستكون في خطوة لاحقة بعد تأكيد نجاح الاستيراد

## بعد الموافقة
بمجرد أن توافق على الخطة، سأبدأ بتنفيذ الاستيراد. توقع أن يستغرق النسخ والتثبيت بضع دقائق حسب حجم المشروع.
